# C2C-B2C_hybrid

objectives -> Consumer-to-Consumer and Business-to-Consumer online marketplace
types of users -> user, verified user (business), admin
priviledges -> as the verified user is always going to be a company, or else saying, only going to be doing B2C commerce, it will be capable of adding the appropriate tags to its products, such as "NEW", whereas the normal user will only have access to "PRISTINE" as
the highest level of product integridy, the verified user will also have a means to be easily identifiable as such by your average user, for example, a colored checkmark next to their username at all times, like the checkmark on Instagram. For now, these are the only
priveledges planned for verified users, maybe exemption from having to make a video of the product when the product is brand-new, comes straight from the factory. Otherwise, procedure is to be followed at all times. admin has the highest level of authority, capable
of adding tags and removing them at all times aswell as remove products and terminate accounts, other priviledges planned. A mod-type user is also planned to be made, with fewer priviledges, but still capable of removing content from the website, granted, if he has
sufficient reason to do so. The mod is also tasked with approving/dissaproving/bargaining user requested tags in the creation of the product planned to be put on sale and dissaproving the creation of a product sale if it doesnt follow protocol, both bargaining, addition and
removal of tags on the product requested to be put for sale will be followed by a notification to the user's notification box where he will be capable of checking out the changes made, followed by an accept and decline boxes where the user can click to accept, thus,
being added to the website or decline, where the user can choose between giving up on the request, or do a counter-proposal to the mod tasked with the user's request. The mod should not outright shut down the request while in the bargaining stage without enough justification
to do so.
